Hands-On Lab: Watson Discovery

 

In this lab you will learn how to build a demo application using Watson Discovery service (and Watson Assistant) that can be used to help demonstrate the value of Discovery for conversational applications.

 

The core use case represented is user support. In many industries, support is essential to providing great experiences, achieving customer satisfaction, and building brand loyalty. Conversational AI solutions using Watson Discovery can help achieve two goals for support organizations:

1.    Deflect support cases away from human agents by providing customers the answer to their question

2.    Assist support agents in finding responses quickly to reduce case handling times and improve overall customer satisfaction

 

This lab presents an example of support in a retail banking setting to help with case deflection. For this use case there are few key benefits to highlight with Discovery:

1.    The ability to answer natural language questions that may come directly from customers that are unfamiliar with an organization’s products

2.    The ability to answer these questions directly from existing content instead of having to spend time and resources creating new content

3.    The ability to train the service to improve how it both reads the content and finds answers to provide better results to the user

Step 2: Train Smart Document Understanding

 

Smart document understanding (SDU) is a new feature in Watson Discovery for learning the structure of documents. It allows you to train on examples of the document structure, and then apply the model to more data, to take something like a pdf and extract structure and meaning. This becomes extremely valuable when trying to create well-formatted, metadata-rich answers within a conversational interface.

 

1.    Select  on the left panel and click on the newly created collection to get to the Overview page of the collection.

2.    Select  from the Overview page.

3.    You will come to the Smart Document understanding labeling screen. This is where you train SDU on your text.

a.    On the right are “Labels” these are the tags that you can assign to different parts of the document. (Note this is training on the structure, so it ignores the words on the page).

b.    In the center you see a view of the original PDF and a view with each section block highlighted, this is where you can make annotations by selecting a label and then a block

c.     On the left are all the pages of the document.

d.    For each page, SDU will show a preview of what the model thinks the content should contain, as you add annotations, you will see these predictions update for future pages.

 

4.    For now let’s skip the title page since it’s a bit unique. Starting on page two, you can see the structure of the document. It has a Title section in blue and then a number of FAQs as subtitles. So we can use the labels to mark the document as such.

5.    There is also a consistent footer on every page, we want to identify that footer so we can potentially remove it.

6.    When done hit submit page, and you’ll advance to the next page in the document.

7.    Let’s mark up one more page and then see how the model is doing.

a.    Customer Value: With a small amount of training you can get a very accurate and scalable model that can be applied to a larger set of documents. Reduces the need for costly preprocessing and curation.

 

8.    Now when we go to the next page we can start to see that the model understands the structure. Without annotating, it has found the subtitles, and footer on the page. 

9.    One of the other major benefits of SDU is the ability to extract data from Tables. Page 40 has an example of table that can be annotated. Use the table label to mark where the table appears in the document. You should see a button appear to “Annotate Table”

10. When annotating the table you mark where the columns and rows appear, then mark the column and row headers. You’ll see a preview of the table appear on the left as you annotate.

 

11. When done with annotation we can hit Submit Table in the top right

a.    Customer Value: The ability to extract structure from tables in unstructured content is a unique differentiator in Discovery and provides the ability to perform complex analysis over docs in more Expert Assist focused solutions (where the end user is a knowledge worker rather than a customer or support agent)

12. Once we’ve gone through and annotated enough pages to feel confident the model is picking up what we need, we can select “Apply Changes to Collection” and it will sync our document again using the SDU model we just produced.

The most common use of SDU is for managing fields within your documents. Each label will appear as a separate field for the doc in Discovery. You can create structured queries to run against these fields.

 

1.    Let’s look at how to query what we got from SDU. To start we’ll run an empty query to see the JSON structure of our document

2.    From the results you can see the document has a subtitle, text, and source.

a.    Customer Value: We can already see that SDU has helped identify the structure of the doc that we can use to create something like FAQs. This reduces the need to produce this structured data separately.

Step 6: Enhancing Results

Now that we have setup our application, we can run some test queries and see how we can potentially improve the results.

Start by running a few sample queries like, “does auto rental coverage apply in other countries” or “is lost luggage covered”

Running these queries will give an initial sense for how well the service is doing

Ideally, you want to create a test set of queries (at least 50) that you can measure the accuracy against. There are several metrics that can be used for measuring accuracy of results from Discovery see here for some suggestions. This set can then be used to determine when you see improvement and help quantify the effort/benefit for customers.

Performing Relevancy Training

1.    One method for improving the order of results is performing Relevancy Training.

a.    Relevancy training provides a supervised machine learning approach for helping Discovery learn which results to pull to the top

b.    For more detail on Relevancy Training, see

                                               i.     Documentation here: https://console.bluemix.net/docs/services/discovery/train-tips.html#relevancy-tips

                                             ii.     Blog post here: https://developer.ibm.com/dwblog/2017/get-relevancy-training/

                                            iii.     Internal FAQ here: https://apps.na.collabserv.com/wikis/home?lang=en-us#!/wiki/W19ab95afacb0_412d_b762_2e874c0f2a9e/page/FAQ%20-%20Relevancy%20Training?section=Relevancy%20Training

c.     Relevancy training can be especially valuable when used in combination with SDU. SDU creates new metadata fields that Relevancy training can then learn over to improve results.

2.    In order to perform relevancy training you need a set of training questions and results. One way to collect examples for training is to use the logging and metrics features in the application

a.    Discovery logs queries run against private data collections and makes them accessible for analytics

b.    To use logged queries for training, select “Train Watson to improve results” from the query results page in the Discovery tooling

c.     From the Training screen, select “Add recent queries” You should see a list of the queries that were run against the collection.

d.    Select queries that are most representative of real customer queries for this use case and then chose “Add to training list”

3.    We can now start rating results, for each question go to “Rate Results”

a.    From the rating screen you will see the top three results for the training query. You can choose whether a result is relevant or not relevant.

                                               i.     Note: It is helpful to provide some examples of Not Relevant results, but it is not necessary to mark all Not Relevant documents, the service treats empty and not relevant similarly.

 

b.    Relevancy Training needs a minimum of 50 training queries with at least one Relevant rating to start.

c.     Customer Value: Relevancy Training enables improvement of results by a business user or content expert, as opposed to many other search systems that require knowledge of search optimization techniques and continuous tuning.

Other Approaches to Improving Relevancy

There are a few other approaches that can provide significant value but are only available in the Advanced plan of Discovery in Public environments:

1.    Custom stopword lists: Stopwords are common terms that contribute little to the overall relevance of a result. Discovery comes with a small list of pre-defined stopwords, but especially in cases where you only have a small number of documents (<1000) it may help to add a broader list of custom stopwords. Thse stopword lists can be added using the Discovery API: https://console.bluemix.net/docs/services/discovery/using.html#stopwords

a.    For English a common list that can be used is here https://gist.github.com/sebleier/554280

 

2.    Continuous Relevancy Training: When building an application, often the best training data can come from real user behavior. Continuous Relevancy Training allows Discovery to use data that comes from logging embedded within your app to improve results automatically as users interact with the service. https://console.bluemix.net/docs/services/discovery/continuous-training.html#crt

a.    Customer Value: Allows a customer to scale relevance without manual intervention. Also helps produce measurable results from the interactions, e.g. average clickthrough rate and the change over time.
